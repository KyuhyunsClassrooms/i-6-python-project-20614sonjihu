# 프로젝트 계획서

## 1. 프로젝트 주제

내가 만들 프로그램의 주제: 사용자의 건강 목적과 연령대에 맞춰 맞춤형 영양제와 주의사항을 안내하는 영양제 추천 프로그램


---

## 2. 문제 상황

이 프로그램이 해결하려는 문제를 설명하세요.

내 설명: 사용자가 자신의 건강 목적(피로회복, 눈 건강 등)과 연령대를 입력하면, 영양제 데이터 중에서 조건에 맞는 맞춤형 영양제와 복용 시 주의사항을  확인할 수 있도록 추천해 주고 싶다


---

## 3. 입력-처리-출력 구조

### 입력

사용자가 입력하거나 프로그램이 사용할 데이터는 무엇인가?

- 사용자 입력:  사용자가 개선하고 싶은 건강 분야를 메뉴에서 골라 입력하는 숫자, 사용자의 나이대 범위
- 프로그램이 사용할 데이터: 영양제 이름, 효능 번호, 추천 연령대, 주의사항
- 

### 처리

입력된 데이터를 어떤 규칙이나 알고리즘으로 처리할 것인가?

- 필터링 알고리즘
- 순서: 사용자 입력 받아두기 - 2차원 리스트 순차 탐색 - 복합 조건 검사 - 결과 출력 및 상태 기록 - 예외 상황 처리 (데이터가 없을 때)
- 규칙1: 리스트의 효능 번호가 사용자가 입력한 목적 번호와 정확히 일치해야 한다.
- 규칙2: 리스트의 추천 연령대가 사용자가 입력한 연령대와 일치하거나, 누구나 먹을 수 있는 전연령이어야 한다.

### 출력

프로그램이 최종적으로 무엇을 출력하는가?

- 사용자가 입력한 건강 목적과 연령대에 부합하는 영양제 이름, 주요 효능, 복용 시 주의사항을 화면에 깔끔하게 출력한다. 조건에 맞는 결과가 없을 때는 예외 안내 메시지를 출력한다.
-

## 6. 사용할 제어구조

### 조건문을 사용할 부분


내 계획:1. 사용자가 입력한 건강 목적 번호와 영양제의 효능 번호가 일치하는지 확인한다.
       2. 사용자가 입력한 연령대와 추천 연령대가 일치하는지 확인한다.
       3. 추천 결과가 없는 경우 안내 메시지를 출력한다.
       4. 잘못된 메뉴 번호를 입력한 경우 오류 메시지를 출력한다.


### 반복문을 사용할 부분

예) 2차원 리스트의 모든 항목을 하나씩 검사한다.

내 계획: 2차원 리스트의 모든 영양제 데이터를 하나씩 검사한다.
        추천 결과가 여러 개일 경우 반복문으로 모두 출력한다.
        사용자가 다시 검색할 수 있도록 프로그램을 반복 실행할 수 있다.
