# 프로젝트 계획서

## 1. 프로젝트 주제

내가 만들 프로그램의 주제: 사용자의 건강 목적과 연령대에 맞춰 맞춤형 영양제와 주의사항을 안내하는 영양제 추천 프로그램


---

## 2. 문제 상황

이 프로그램이 해결하려는 문제를 설명하세요.

내 설명: 사용자가 자신의 건강 목적(피로회복, 눈 건강 등)과 연령대를 입력하면, 영양제 데이터 중에서 조건에 맞는 맞춤형 영양제와 복용 시 주의사항을  확인할 수 있도록 추천해 주고 싶다


---

## 3. 입력-처리-출력 구조

### 입력

사용자가 입력하거나 프로그램이 사용할 데이터는 무엇인가?

- 사용자 입력:  사용자가 개선하고 싶은 건강 분야를 메뉴에서 골라 입력하는 숫자, 사용자의 나이대 범위
- 프로그램이 사용할 데이터: 영양제 이름, 효능 번호, 추천 연령대, 주의사항
- 

### 처리

입력된 데이터를 어떤 규칙이나 알고리즘으로 처리할 것인가?

- 필터링 알고리즘
- 순서: 사용자 입력 받아두기 - 2차원 리스트 순차 탐색 - 복합 조건 검사 - 결과 출력 및 상태 기록 - 예외 상황 처리 (데이터가 없을 때)
- 규칙1: 리스트의 효능 번호가 사용자가 입력한 목적 번호와 정확히 일치해야 한다.
- 규칙2: 리스트의 추천 연령대가 사용자가 입력한 연령대와 일치하거나, 누구나 먹을 수 있는 전연령이어야 한다.

### 출력

프로그램이 최종적으로 무엇을 출력하는가?

- 사용자가 입력한 건강 목적과 연령대에 부합하는 영양제 이름, 주요 효능, 복용 시 주의사항을 화면에 깔끔하게 출력한다. 조건에 맞는 결과가 없을 때는 예외 안내 메시지를 출력한다.
-

## 4. 2차원 리스트 설계

내 프로그램에서 사용할 2차원 리스트의 이름:  supplements


```python
supplements = [
    ["비타민B군", 1, "전연령", "공복에 먹으면 속이 쓰릴 수 있으니 식후 복용을 권장합니다."],
    ["밀크씨슬", 1, "20~50대", "알레르기 체질인 경우 성분을 확인하고 복용하세요."],
    ["루테인", 2, "20~50대", "과다 섭취 시 피부가 황색으로 변할 수 있으니 하루 한 알만 드세요."],
    ["오메가3", 2, "전연령", "혈액 응고 억제 작용이 있으므로 수술 전에는 복용을 피하세요."],
    ["유산균", 3, "전연령", "항생제와 함께 복용할 때는 최소 2시간 이상 간격을 두세요."],
    ["칼슘 & 비타민D", 4, "60대 이상", "신장 질환이 있는 경우 복용 전 의사와 상의해야 합니다."]
]
```

### 한 행의 의미


### 각 열의 의미

0번 열: 추천 영양제의 이름을 의미합니다. (예시: "비타민B군", "오메가3")

1번 열: 개선하고 싶은 건강 목적 번호를 정수형태로 나타냅니다. (예시: 1은 피로 회복, 2는 눈 건강)

2번 열: 영양제 복용을 권장하는 대상 연령대를 의미합니다. (예시: "20~50대", "전연령")

3번 열: 해당 영양제를 먹을 때 조심해야 하는 복용 시 주의사항을 의미합니다. (예시: "공복에 먹으면 속이 쓰릴 수 있으니...")

## 5. 함수 설계

display_menu 함수

역할: 사용자에게 화면으로 선택할 수 있는 4가지 건강 목적 메뉴(피로 회복, 눈 건강 등)를 보여줍니다.

입력값: 없음

반환값 또는 출력: 없음 (화면에 메뉴 텍스트만 출력함)

get_user_input 함수

역할: 사용자로부터 원하는 목적 번호(숫자)와 현재 연령대(글자)를 입력받아 프로그램에 전달합니다.

입력값: 없음

반환값 또는 출력: 사용자가 입력한 목적 번호(purpose)와 연령대(age)를 반환함

recommend_supplement 함수

역할: 2차원 리스트를 처음부터 끝까지 돌면서 사용자가 입력한 조건과 일치하는 영양제 이름과 주의사항을 찾아 화면에 출력합니다. 만약 맞는 데이터가 하나도 없다면 안내 예외 메시지를 보여줍니다.

입력값: 영양제 데이터(data), 목적 번호(purpose), 연령대(age)

반환값 또는 출력: 없음 (화면에 추천 결과 텍스트를 출력함)

main 함수

역할: 위의 세 가지 함수(display_menu -> get_user_input -> recommend_supplement)를 순서대로 실행하여 프로그램이 멈추지 않고 매끄럽게 돌아가도록 전체 흐름을 제어합니다.

입력값: 없음

반환값 또는 출력: 없음

## 6. 사용할 제어구조

### 조건문을 사용할 부분


내 계획:1. 사용자가 입력한 건강 목적 번호와 영양제의 효능 번호가 일치하는지 확인한다.
       2. 사용자가 입력한 연령대와 추천 연령대가 일치하는지 확인한다.
       3. 추천 결과가 없는 경우 안내 메시지를 출력한다.
       4. 잘못된 메뉴 번호를 입력한 경우 오류 메시지를 출력한다.


### 반복문을 사용할 부분

2차원 리스트의 모든 항목을 하나씩 검사한다.

내 계획: 2차원 리스트에 저장된 모든 영양제 데이터를 처음부터 끝까지 하나씩 검사하여, 사용자의 선택 조건에 맞는 추천 영양제가 여러 개일 경우 반복문으로 모두 찾아내어 화면에 출력한다.
