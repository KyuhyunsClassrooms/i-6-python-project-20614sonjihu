# 최종 보고서

## 1. 프로그램 주제와 목적

내 프로그램의 주제: 사용자의 건강 목적과 연령대에 맞춰 맞춤형 영양제와 주의사항을 안내하는 영양제 추천 프로그램

이 프로그램을 만든 이유: 진로과 화학, 생명 분야이고, 여러 증상이 있어있을때 어떤 영양제를 먹어야할지 도움을 주기 위해 만들었다.


---

## 2. 문제 분해 및 모델링

### 입력
사용자가 개선하고 싶어 하는 건강 목적 번호를 정수형(int)으로 입력받는다. ( 1~4 사이의 숫자)

사용자의 현재 연령대를 문자열(str) 형태로 입력받는다. ("10대", "20~50대", "60대 이상")

### 처리
2차원 리스트 탐색: 저장된 영양제 데이터(supplements 2차원 리스트)를 for 반복문을 사용하여 한 행(row)씩 처음부터 끝까지 순회하며 검사한다

조건 검사 및 매칭: 사용자가 입력한 목적 번호가 리스트의 목적 번호(1번 열)와 일치하고, 입력한 연령대가 리스트의 연령대(2번 열)와 일치하거나 '전연령'인 경우를 if 조건문과 논리 연산자(and, or)로 판단한다

상태 변수(Flag) 관리: 조건에 맞는 영양제를 찾으면 found 변수의 값을 True로 변경하여 매칭 성공 상태를 기록합니다. 만약 반복문이 끝날 때까지 조건에 맞는 데이터를 찾지 못하면(if not found), 추천 데이터가 없다는 예외 처리를 수행한다

### 출력
사용자의 조건과 일치하는 추천 영양제의 이름(0번 열)과 해당 영양제의 복용 시 주의사항(3번 열)을 화면에 깔끔한 문장 형식으로 출력한다.

일치하는 데이터가 없을 경우, "선택하신 조건에 맞는 추천 영양제가 데이터에 없습니다."라는 예외 안내 메시지를 화면에 출력한다.

## 3. 사용한 2차원 리스트 설명

2차원 리스트 이름: supplements

한 행의 의미: 영양제 한 종류가 가진 전체 정보(이름, 목적 번호, 권장 연령대, 복용 주의사항)를 나타낸다.

각 열의 의미:  

0번 열: 추천 영양제의 이름을 의미한다. (실제 예시: "비타민B군", "오메가3")

1번 열: 개선하고 싶은 건강 목적 번호를 정수형태로 의미한다. (실제 예시: 1, 2)

2번 열: 영양제 복용을 권장하는 대상 연령대를 의미한다. (실제 예시: "20~50대", "전연령")

3번 열: 해당 영양제를 먹을 때 조심해야 하는 복용 시 주의사항을 의미한다. (실제 예시: "공복에 먹으면 속이 쓰릴 수 있으니...")


이 2차원 리스트를 사용한 이유:

1. 영양제 한 종류에는 이름뿐만 아니라 목적, 연령대, 주의사항 등 여러 개의 연관된 데이터가 함께 묶여 있어야 한다.

2. 만약 일반 리스트를 썼다면 이 데이터들을 서로 연결하기 복잡했겠지만, 2차원 리스트를 사용함으로써 영양제 하나당 하나의 행(row)으로 묶어 체계적이고 효율적으로 관리할 수 있기 때문이다.

3. 또한 for 반복문을 이용해 많은 양의 영양제 데이터를 한 줄씩 순서대로 꺼내며 사용자의 조건과 비교하기에 매우 적합한 자료구조이기 때문에 사용했다.

## 4. 함수 설명

1. display_menu 함수

역할: 사용자에게 화면으로 선택할 수 있는 4가지 건강 목적 메뉴를 출력합니다.

입력값: 없음

반환값 또는 출력: 없음 (화면 텍스트 출력)

2. get_user_input 함수

역할: 사용자로부터 원하는 목적 번호와 연령대를 입력받아 변수에 저장하고 이를 전달합니다.

입력값: 없음

반환값 또는 출력: purpose (정수형 목적 번호), age (문자열 연령대) 반환

3. recommend_supplement 함수

역할: 2차원 리스트를 탐색하며 조건에 맞는 영양제 정보를 출력하고, 없으면 예외 메시지를 보여줍니다.

입력값: data (영양제 2차원 리스트), purpose (목적 번호), age (연령대)

반환값 또는 출력: 없음 (화면 텍스트 출력)

4. main 함수

역할: 메뉴 표시, 입력 받기, 추천 함수를 순서대로 호출하여 프로그램의 전체 흐름을 제어합니다.

입력값: 없음

반환값 또는 출력: 없음



가장 중요한 함수 하나를 골라 자세히 설명하세요.

함수 이름: recommend_supplement

설명:
    이 함수는 프로그램의 핵심 알고리즘인 '데이터 검색 및 예외 처리'를 담당한다.

매개변수로 영양제 데이터가 담긴 2차원 리스트(data)와 사용자가 입력한 값(purpose, age)을 전달받는다. 함수가 시작되면 조건에 맞는 영양제를 찾았는지 기억하기 위해 상태 변수인 found를 False로 초기화한다.

그 후 for row in data: 반복문을 통해 2차원 리스트의 행을 하나씩 꺼내며 검사한다. 이때 if 조건문을 사용하여 사용자의 목적 번호가 리스트의 1번 열과 일치하는지, 그리고 사용자의 연령대가 리스트의 2번 열과 일치하거나 '전연령'인지를 판단한다. 조건이 일치하면 영양제 이름과 주의사항을 화면에 출력하고, found 변수를 True로 변경하여 매칭 성공 상태를 기록한다.

마지막으로 리스트를 모두 순회했음에도 found 변수가 여전히 False라면, 입력된 조건에 맞는 데이터가 존재하지 않는다는 의미이므로 예외 메시지를 출력하여 안전하게 예외를 처리한다.

## 5. 조건문과 반복문 활용

### 조건문 사용 부분

어떤 조건을 검사했는가?

:recommend_supplement 함수 내에서 2차원 리스트(data)의 각 행을 검사할 때, 사용자가 입력한 건강 목적 번호(purpose)가 리스트의 목적 번호(1번 열)와 같은지 검사한다.

동시에 사용자가 입력한 연령대(age)가 리스트의 연령대(2번 열)와 일치하거나, 혹은 누구에게나 해당하는 '전연령'이라는 값을 가졌는지 검사한다.

또한, 반복문이 완전히 끝난 후 found 변수가 False 유지 여부를 검사하여 조건에 맞는 영양제가 하나도 없었는지 판단한다.


### 반복문 사용 부분

무엇을 반복 처리했는가?

for row in data: 반복문을 사용하여 영양제 정보가 담긴 2차원 리스트의 전체 데이터를 첫 번째 행부터 마지막 행까지 한 줄씩 순서대로 꺼내는 과정을 반복 처리한다.


### 조건문과 반복문이 함께 작동한 부분


for 반복문이 2차원 리스트를 순회하며 영양제 데이터를 하나씩 꺼내오면, 그 반복문 내부에서 if 조건문이 작동하여 사용자의 입력값과 현재 데이터가 일치하는지 비교·분석한다.

조건이 일치할 때마다 영양제 정보와 주의사항을 화면에 반복하여 출력함으로써, 하나의 건강 목적에 해당하는 여러 개의 추천 영양제를 누락 없이 모두 찾아내어 보여주는 역할을 함께 수행한다.

## 7. 테스트와 오류 수정

### 테스트 1: 정상 사례

입력: 건강 목적 번호: 1 (피로 회복)

연령대: 20~50대


예상 결과: 

2차원 리스트(supplements)에서 목적 번호가 1이면서 연령대가 20~50대인 '밀크씨슬' 데이터와, 연령대가 '전연령'인 '비타민B군' 데이터가 동시에 매칭되어야 한다.

따라서 화면에 두 영양제의 이름과 각각의 복용 주의사항이 에러 없이 차례대로 출력되어야 한다.

실제 결과:

예상했던 바와 같이 '비타민B군'과 '밀크씨슬'의 추천 결과 및 주의사항 텍스트가 정상적으로 화면에 출력된다.

프로그램이 중간에 비정상 종료되지 않고 안내선(---)과 함께 매끄럽게 실행을 종료함을 확인한다.


### 테스트 2: 예외 또는 경계 사례

입력: 건강 목적 번호: 4 (뼈 / 관절 건강)

연령대: 10대

예상 결과: 

 현재 2차원 리스트(supplements) 데이터 중에서 목적 번호가 4인 영양제는 '60대 이상' 조건만 존재한다. 따라서 10대라는 입력값과 매칭되는 데이터는 없어야 한다.

프로그램은 데이터를 찾지 못하더라도 오류(NameError 등)를 내며 강제 종료되지 않고, "선택하신 조건에 맞는 추천 영양제가 데이터에 없습니다."라는 예외 안내 메시지를 화면에 정상적으로 출력해야 한다.

실제 결과: 

2차원 리스트 순회가 끝난 후 found 변수가 False인 상태를 정확히 인식하여, 개발 시 설계했던 예외 처리 안내 문구가 화면에 정상적으로 출력된다.

잘못된 조건 입력 상황에서도 시스템이 다운되지 않고 안전하게 예외 처리를 수행함을 확인한다.




### 발견한 오류와 수정한 방법

프로그램 실행 중 recommend_supplement 함수 내부에서 조건에 맞는 영양제를 찾았을 때 실행되는 코드 줄(line)에서 정의되지 않은 빈칸 기호(_________)가 그대로 실행되면서 NameError: name '_________' is not defined라는 명칭 오류가 발생하며 프로그램이 강제 종료되는 문제를 발견하였다. 이 오류의 원인은 파이썬 문법에서 인식할 수 없는 임의의 기호를 컴퓨터가 변수나 함수 이름으로 오인했기 때문이며, 이를 해결하기 위해 영양제를 성공적으로 찾았을 때 프로그램이 그 상태를 기억할 수 있도록 빈칸 기호를 파이썬의 불리언(Boolean) 예약어인 참(True)으로 대입하여 found = True 형태로 코드를 수정하였다. 결과적으로 2차원 리스트 내부 탐색 과정에서 일치하는 데이터가 발견되면 found 변수의 값이 정상적으로 업데이트되어 프로그램이 중간에 튕기지 않게 되었으며, 이후 진행되는 예외 처리 조건문(if not found:)도 에러 없이 의도한 논리대로 올바르게 작동함을 확인하였다.

## 8. Gemini 활용 내용 요약



Gemini에게 도움받은 부분:

이번 프로젝트를 수행하는 과정에서 학교 계정 Gemini를 인공지능 코치로 적극 활용하여 프로그램의 완성도를 높였다. 초기 단계에서는 증상에 따른 영양제 추천 프로그램의 실행 과정 중 발생한 NameError 오류의 원인을 함께 분석하고, 빈칸 코드를 파이썬 불리언 예약어인 True로 수정하는 디버깅 도움을 받아 프로그램의 핵심 알고리즘을 안전하게 완성하였다. 이후 학교 수행평가의 필수 조건인 2차원 리스트 활용, 함수 분리, 제어문 사용 등의 루브릭 기준을 만족하는지 코드를 교차 검증받았으며, project_plan.md와 report.md 작성에 필요한 데이터 구조 분석 및 함수 설계 내용을 보고서 형식에 알맞은 줄글 형태로 정제하는 데 도움을 받았다. 결과적으로 AI와의 지속적인 피드백을 통해 코드의 문법적 오류를 해결했을 뿐만 아니라, 개발 계획과 수행 과정을 논리적인 문서로 기록하고 구조화하는 역량을 기를 수 있었다.


Gemini가 제안했지만 내가 수정하거나 사용하지 않은 부분:  

사용자가 종료를 원할 때까지 프로그램을 무한 반복하여 재실행할 수 있도록 메인 함수 내부에 while 무한 루프와 break문을 추가하는 구조를 제안받았다. 하지만 현재 main.py 소스코드가 학교에서 요구하는 기본 기능에 충실하게 잘 작동하고 있으므로, 복잡성을 줄이고 안정적인 제출을 위해 이번 최종 코드에는 해당 기능 연장 제안을 반영하지 않고 보류하였다.


최종 코드에서 내가 직접 판단하거나 수정한 부분:

제안받은 여러 가지 설명 문구 중에서 예시 양식을 그대로 복사하는 대신, 내가 만든 영양제 데이터 검색 방식과 조건문 알고리즘의 실제 작동 원리에 맞추어 보고서 내용을 직접 수정하고 대입하였다. 또한 무한 반복 실행 기능(while문)을 무조건 추가하기보다 현재 제출용 코드의 깔끔한 마무리가 더 중요하다는 것을 스스로 판단하여 기존의 안정적인 함수 구조를 유지하는 방향으로 최종 코드를 확정하였다.

## 9. 한계와 개선 방안

현재 프로그램의 한계:

현재 프로그램은 코드가 실행될 때 미리 소스코드 내부에 고정해 둔 2차원 리스트 데이터만 조회할 수 있다는 한계가 있다. 이로 인해 새로운 영양제 정보를 추가하거나 기존 데이터를 수정하려면 프로그램 코드 자체를 직접 변경해야 하므로 일반 사용자가 사용하기에 불편함이 있다. 또한, 한 번의 추천 과정이 끝나면 프로그램이 즉시 종료되기 때문에, 사용자가 다른 건강 목적이나 연령대를 연속해서 검색하고 싶을 때 프로그램을 매번 다시 실행해야 하는 번거로움이 존재한다.


다시 개선한다면 추가하고 싶은 기능:

이러한 한계를 극복하기 위해 향후 프로그램을 다시 개선한다면, 사용자로부터 키보드로 새로운 영양제 데이터를 직접 입력받아 2차원 리스트에 실시간으로 행(row)을 추가하는 데이터 입력 기능을 구현하고 싶다. 더불어 앞서 구상했던 대로 while 반복문과 사용자 확인 유도 조건문을 결합하여, 사용자가 종료를 원할 때까지 프로그램을 끊김 없이 연속해서 재실행할 수 있는 다중 검색 및 프로그램 제어 기능을 추가하고 싶다.
